# Projekt: Grammat (recept + inköpslista)

Personlig receptsajt för Patrik + vänner. **Live:** https://orgutveckling.se/recept/
Repo: https://github.com/Elwyndaz/recept (GH Pages, branch `main`, root). Custom-domänen ärvs från elwyndaz.github.io, ingen CNAME här.

## Funktion (as built)
- Receptbibliotek och inköpslista är UNIKA per användare. Nya konton/anonyma besökare börjar med tomt `state.recipes: []`. `starter.json` seedar INTE längre personlig state, det är bara grunddata som alltid visas i fliken "Allas recept" (tillsammans med andra inloggade användares recept, hämtade via `GET /allas-recept`); användaren väljer själv vad som ska in i "Mina recept" via "Lägg till"-knappen där. Recept läggs till/redigeras/tas bort i UI:t, INTE via repot.
- Konto = användarnamn + PIN (inga mejl). Token (uuid) sparas i localStorage. PIN kan inte återställas i UI, reset görs manuellt i D1.
- Alla ingrediensmängder kanoniskt i g eller ml så att listan kan summera över recept. `count`/`countUnit` ger ungefärligt styckantal i visningen ("220 g (~2 st)"). `toTaste: true` = "efter smak" (ingen mängd). `skipList: true` = ingrediens (t.ex. vatten) utesluts ur inköpslistan. `group` = rubrik inom receptet (t.ex. "Sås"). `cat` = varukategori: grönt/kött/mejeri/skafferi/fryst/övrigt.
- Portionsskalning linjärt mot `portions` (basportioner). Summeringsnyckel = ingrediensnamn lowercase/trim, så samma vara MÅSTE stavas lika mellan recept för att slås ihop.
- Varje recept har en `course` (Förrätt/Huvudrätt/Efterrätt/Drycker/Såser & röror). Katalogvyn grupperar recepten under rubriker per `course`, fast ordning, tomma grupper visas inte. Saknas/ogiltig course: faller tillbaka till Huvudrätt.
- Inköpslistan är formgiven som ett butikskvitto (signaturelementet): mono-typsnitt, kategorirubriker, avbockning, källrecept per rad (tap på namnet), egna extrarader, "Töm listan".
- Backup finns under Konto: laddar ner hela state som `grammat-backup-YYYY-MM-DD.json` och kan återställa samma format senare. Backupen innehåller recept, val, extrarader och avbockningar. Vid inloggat konto synkas återställd backup vidare till D1 via ordinarie save.
- Serverbackup utan kort/R2: D1 Time Travel är den automatiska korttidsbackupen. För stående agentbackup ska Codex/Claude exportera D1 till lokal `backups/` innan riskabla ändringar, D1-migreringar eller deploy. `backups/` är git-ignored och ska inte pushas eller publiceras på GitHub Pages.

## Filer
- `index.html`: all CSS inline, design tokens i `:root`. Typsnitt: Bricolage Grotesque (display), Schibsted Grotesk (brödtext), Spline Sans Mono (siffror/kvitto). Palett: papper #EAEDDF, ink #20281A, lingon #9E2B3E, senap #DDA321, kvitto #FBFAF2. Mobil: fast bottennav; desktop (≥720px): nav under ordmärket. `.wrap` max-width 640px, `.cards` 2 kolumner (≥720px); vid ≥1000px breddas `.wrap` till 920px och `.cards` blir 3 kolumner. `.card` är `flex-column` med `.card-row{margin-top:auto}` så kortknapparna bottenankrar oavsett titel-radlängd. noindex.
- `app.js`: rena funktioner överst (aggregate, fmtNum, fmtItem, fmtIngredient, spiceHint, slugify, parseImport, normalizeState, makeBackup — exporteras för test). `spiceHint(amount, unit, cat)` gissar krm/tsk/msk (densitet ≈1 g/ml) för ovägda skafferivaror ≤30 g utan `count` satt, visas bredvid grammängden i både receptvy och inköpslista - ren display, ingen datamigrering. Receptvyn har en "Kopiera receptet"-knapp (`data-duplicate`) som djupklonar receptet med nytt id/titel (" (kopia)") och öppnar redigeraren. AI-import: vyn #/importera visar en kopierbar prompt (AI_PROMPT i app.js) som får valfri AI-modell att svara med en JSON-array av recept i exakt rätt format (flera recept/länkar på en gång går bra); parseImport tar objekt eller array, returnerar alltid array, allt-eller-inget vid fel ("Recept 2: ..."), id-krockar inom batchen får suffix; 1 importerat recept → receptvyn, flera → Mina recept. SPA:n (hash-routing: #/ Mina recept, #/recept/:id, #/nytt, #/redigera/:id, #/importera, #/lista, #/allas Allas recept, #/konto). #/allas visar starter.json grupperat per `course` (alltid, även utloggad) plus, om inloggad, andra användares recept (hämtade från `/allas-recept`, filtrerade mot starter-id:n client-side); "Lägg till i mina recept"-knapp per kort, ersätts av "Ta bort ur mina recept" (`data-remove-allas`) om id redan finns i din `state.recipes`, för snabb bulk add/remove utan att öppna receptet. State = `{recipes, selections, extras, checked, struck}` i localStorage; PUT:as hel (debounce 800 ms) till API:t när man är inloggad; vid inloggning/sidladdning hämtas serverns state (servern vinner om den har data, annars laddas lokalt upp). Sista skrivning vinner. `struck = {receptId: [ingrediensnyckel]}`: i receptvyn (egna recept) togglar tap på en ingrediensrad "har hemma/redan i grytan", raden stryks över (senapsbock, flyttas sist, animeras med View Transitions API där det finns) och varan utesluts ur inköpslistan (aggregate tar struck som tredje argument); nollas när receptet tas bort ur listan (även Töm listan/radering). Receptkorten är helklickbara (`data-card`, knappraden `.card-row` är död zon). Skärmen hålls vaken i receptvyn via Screen Wake Lock API (syncWakeLock i app.js, ingen UI).
- `starter.json`: 22 startrecept, konverterade till g/ml (1 msk=15 ml, 1 tsk=5 ml, 1 dl=100 ml + vikttabeller). 16 av dem importerade från `recept 2.mht` eller enskilda källor 2026-07-07; några saknar nedskrivna steg (källan var bara video/länk) och visas som "Inga steg nedskrivna". Visas alltid i "Allas recept", oavsett inloggning. Nya recept i repot syns direkt där för alla, ingen per-konto-migrering behövs längre.
- `nutrients.json`: global uppslagstabell ingrediensnamn (samma nyckel som `keyOf` i app.js) → kcal/protein/kolhydrater/fett per 100 g/ml, 97 av 137 baserade på riktiga Livsmedelsverket-värden (från exportfilen på soknaringsinnehall.livsmedelsverket.se, matchad manuellt mot ingrediensnamn 2026-07-07; deras REST-API är trasigt server-side, se `livsmedelsverket-api-research.md`; hela filen genomgången igen 2026-07-07 för att fylla 27 saknade ingredienser och rätta felaktiga uppskattningar, 14 av dem hade riktiga data i exporten), ytterligare 2 (mascarpone, rödpesto) från ICA:s produktsidor (saknades i Livsmedelsverkets databas), resten (kryddblandningar, märkesvaror, drycker utan träff) generella uppskattningar eftersom de saknas överallt. Källa länkad i receptvyn.. `nutritionPerPortion(recipe, nutrients)` i app.js räknar ut värde per portion direkt från receptets grundmängder (oberoende av vald portionsskala), `ml` behandlas som `g`. Ingrediens som saknas i filen räknas inte med, visas som "ofullständigt" i receptvyn. Växer manuellt vid nya ingredienser, samma underhållsmönster som nya startrecept.
- `test.js`: `node test.js` — kontrollerar summering, skalning, skipList, efter smak, svenskt talformat. Kör vid ändringar i app.js/starter.json.
- `worker/`: Cloudflare Worker `recept-api` på **recept-api.orgutveckling.se** (custom domain). OBS: separat från `orgutveckling-votes` på api.orgutveckling.se (rör ej den). D1-databas `recept`, id `519f743c-9c5e-48c0-930b-c615ada71f0c`, en tabell: `users(id, name UNIQUE, pin_hash, token, state)`. `pin_hash` = `salt:sha256hex(salt+pin)`. Endpoints: POST /register, POST /login → `{token,name}`; GET/PUT /state (Bearer token), PUT max 256 kB och kräver `recipes`-array; GET /allas-recept (Bearer token, valfritt vilket konto) → flat lista `[{...recipe, owner}, ...]` från alla konton, ingen dedup/filtrering server-side.

## Kommandon
- Deploy worker: `cd worker && npx wrangler deploy`
- D1-konsol: `npx wrangler d1 execute recept --remote --command "..."`.
- PIN-reset (behåller receptdata): räkna ut ny `salt:hash` lokalt med `node -e "const c=require('crypto');const salt=c.randomUUID();const h=c.createHash('sha256').update(salt+'NYPIN').digest('hex');console.log(salt+':'+h)"`, kör sedan `UPDATE users SET pin_hash='<salt:hash>' WHERE name='användarnamn'` mot D1. Token och state påverkas inte. Använd ENDAST DELETE + återregistrering om kontot är trasigt på annat sätt, det raderar all state om den inte kopierats ut först.
- Agentbackup före riskabla ändringar/deploy: `New-Item -ItemType Directory -Force backups; npx wrangler d1 export recept --remote --output backups/recept-YYYY-MM-DD-HHMMSS.sql`. Ersätt timestamp med aktuell tid. Kontrollera att filen hamnar i `backups/`, inte i repo-roten.
- D1 Time Travel restore: `npx wrangler d1 time-travel info recept --timestamp "YYYY-MM-DDTHH:MM:SSZ"` och `npx wrangler d1 time-travel restore recept --timestamp "YYYY-MM-DDTHH:MM:SSZ"`.
- Frontend deployas genom push till `main` (GH Pages, ~30 s).
- Lokal test: `python -m http.server` i repo-roten (API:t är CORS-öppet).
